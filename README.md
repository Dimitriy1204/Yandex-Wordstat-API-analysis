# WordstatAgent — Yandex Wordstat Analysis Agent v5

Веб-приложение на **FastAPI** для сравнительного анализа поискового спроса через **Yandex Wordstat API**.

Сбор помесячной частотности → расчёт метрик → прогноз **Prophet** → форматированный **Excel-отчёт** → опционально **YandexGPT** → отправка в Telegram / VK / Email.

Разработано для [РИФЕЙ — Завод Стройтехника](https://v-press.ru).

---

## Возможности

| Блок | Описание |
|------|----------|
| **Wordstat** | Помесячная частотность запросов (Yandex Search API v2, GetDynamics) |
| **Метрики** | Рост (4 варианта), волатильность, сезонность, индекс конкуренции |
| **Ниши** | «Голубые океаны» и «Растущие рынки» — по 3 метрике роста, **все сравниваемые запросы** |
| **Prophet** | Прогноз на 6 месяцев (при ≥12 точках истории) |
| **Excel** | 7 листов данных + лист «О файле»; оформление по образцу `yandex_analysis_exemple.xlsx` |
| **Графики** | В Excel — нативные диаграммы; в браузере — Chart.js (топ-N и все запросы) |
| **YandexGPT** | Опционально: расширение запросов, обзор рынка, рекомендации, вердикты по нишам |
| **Доставка** | Скачивание XLSX, отправка в Telegram, VK, Email |
| **Безопасность** | Настройки и пароли хранятся в зашифрованном `settings.enc` (AES-256) |

## Листы Excel-отчёта

| Лист | Содержание |
|------|------------|
| **О файле** | Выводы, методология, блоки YandexGPT (если включён LLM), описание листов |
| **Данные** | Сырые помесячные данные по каждому запросу |
| **Сводка по запросам** | Итоговые метрики, прогноз средних показов, цветовая индикация |
| **Прогноз** | Таблица Prophet на 6 месяцев с доверительными интервалами |
| **График** | Две редактируемые линейные диаграммы Excel (топ-N и все запросы) |
| **Голубые океаны** | 3 блока: низкая конкуренция + рост (momentum / mean / slope) |
| **Растущие рынки** | 3 блока: рейтинг по росту (momentum / mean / slope) |

Имя файла результата: `yandex_analysis_ГГГГММДД_ЧЧММСС.xlsx` (в папке с exe или проектом).

## YandexGPT — опционально

Переключатель **«Использовать LLM»** на главной странице **по умолчанию выключен**.

| Режим | Поведение |
|-------|-----------|
| LLM **выкл.** | Полный отчёт только на данных Wordstat, без блоков YandexGPT |
| LLM **вкл.** + ключи GPT | Расширение запросов, обзор, рекомендации, вердикты нейросети |
| LLM **вкл.** + только Wordstat | Ключ и Folder ID Wordstat подставляются для GPT; нужна роль `ai.languageModels.user` |

Отдельные поля YandexGPT в настройках необязательны — можно оставить пустыми и использовать ключ Wordstat.

Проверка GPT: **Настройки → YandexGPT → «Проверить подключение»**.

```bash

```

```bash

```

Если Prophet не установится, анализ продолжит работу без прогноза.

## Запуск

### Python

```bash
python run.py
```

Откроется **http://127.0.0.1:8000** (браузер запускается автоматически).

Другой порт:

```bash
set PORT=8080
python run.py
```

Доступ в локальной сети:

```bash
set HOST=0.0.0.0
python run.py
```

### Готовый exe

После сборки запустите:

```
dist\yandex_wordstat_agent.exe
```

Достаточно **одного файла exe** — шаблон Excel и интерфейс встроены внутрь. Отдельный `yandex_analysis_exemple.xlsx` рядом с exe не нужен.

---

## Сборка exe

```powershell
powershell -ExecutionPolicy Bypass -File .\build.ps1
```

Скрипт:
1. Останавливает старые процессы `yandex_wordstat_agent.exe`
2. Собирает `dist\yandex_wordstat_agent.exe` через PyInstaller
3. Вкладывает `yandex_analysis_exemple.xlsx` в exe (если файл есть в корне проекта)

Перед повторной сборкой закройте запущенный агент или снова используйте `build.ps1`.

## Использование

1. Откройте веб-интерфейс (`http://127.0.0.1:8000` или после запуска exe).
2. **Настройки** → укажите **API-ключ** и **Folder ID** Yandex Cloud (Wordstat).
3. При необходимости настройте Telegram / VK / Email / прокси / YandexGPT.
4. На главной:
   - введите запросы (через запятую или с новой строки);
   - выберите период;
   - при необходимости включите **«Использовать LLM»**;
   - нажмите **«Запустить анализ»**.
5. Скачайте Excel или отправьте отчёт в выбранный канал.

Расширение запросов от YandexGPT (если включён LLM) записывается в Excel **построчно** — можно скопировать и вставить в поле запросов.

## API-ключи и роли

| Сервис | Где получить | Роль / примечание |
|--------|--------------|-------------------|
| **Wordstat** | [console.cloud.yandex.ru](https://console.cloud.yandex.ru) | Сервисный аккаунт, роль `search-api.executor`, API-ключ, Folder ID |
| **YandexGPT** | Тот же каталог | Роль `ai.languageModels.user`; модели: `yandexgpt-lite`, `yandexgpt`, `yandexgpt-pro` |
| **Telegram** | [@BotFather](https://t.me/BotFather) | Token бота + chat_id |
| **VK** | VK API | access_token (docs, messages) |
| **Email** | SMTP | Например Gmail: `smtp.gmail.com:587` |

## Ограничения

- **Wordstat API** — лимиты тарифа Yandex Cloud (на бесплатном уровне — порядка 100 запросов/сутки).
- **Период** — до 3 лет (ограничение API).
- **Prophet** — нужно ≥12 месяцев истории по запросу.
- **Запросы без данных** — попадают в сводку и таблицы ниш с нулевыми метриками.
- **Прокси** — настраивается в разделе настроек при необходимости.

## Устранение неполадок

| Проблема | Решение |
|----------|---------|
| Exe не пересобирается | Закройте агент; запустите `build.ps1` или `scripts\stop_agent.ps1` |
| Exe не стартует на другом ПК | Установите [VC++ Redistributable x64](https://aka.ms/vs/17/release/vc_redist.x64.exe) |
| «Настройки API не заполнены» | Укажите ключ и Folder ID Wordstat в настройках |
| Нет прогноза | Мало истории (<12 мес.) или Prophet недоступен |
| YandexGPT не отвечает | Проверьте роль `ai.languageModels.user`; кнопка «Проверить» в настройках |
| Порт занят | Закройте старый экземпляр; при запуске exe предыдущий процесс на порту завершается автоматически |
