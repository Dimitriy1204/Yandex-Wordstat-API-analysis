# WordstatAgent — Yandex Wordstat Analysis Agent v5

Веб-приложение на **FastAPI** для сравнительного анализа поискового спроса через **Yandex Wordstat API**.

Сбор помесячной частотности → расчёт метрик → прогноз **Prophet** → форматированный **Excel-отчёт** → опционально **YandexGPT** → отправка в Telegram / VK / Email.

Разработано для [РИФЕЙ — Завод Стройтехника](https://v-press.ru).

---

## Возможности

| Блок | Описание |
|------|----------|
| **Wordstat** | Помесячная частотность запросов (Yandex Search API v2, GetDynamics) |
| **Метрики** | Рост (4 варианта), волатильность, сезонность, индекс конкуренции |
| **Ниши** | «Голубые океаны» и «Растущие рынки» — по 3 метрике роста, **все сравниваемые запросы** |
| **Prophet** | Прогноз на 6 месяцев (при ≥12 точках истории) |
| **Excel** | 7 листов данных + лист «О файле»; оформление по образцу `yandex_analysis_exemple.xlsx` |
| **Графики** | В Excel — нативные диаграммы; в браузере — Chart.js (топ-N и все запросы) |
| **YandexGPT** | Опционально: расширение запросов, обзор рынка, рекомендации, вердикты по нишам |
| **Доставка** | Скачивание XLSX, отправка в Telegram, VK, Email |
| **Безопасность** | Настройки и пароли хранятся в зашифрованном `settings.enc` (AES-256) |

## Листы Excel-отчёта

| Лист | Содержание |
|------|------------|
| **О файле** | Выводы, методология, блоки YandexGPT (если включён LLM), описание листов |
| **Данные** | Сырые помесячные данные по каждому запросу |
| **Сводка по запросам** | Итоговые метрики, прогноз средних показов, цветовая индикация |
| **Прогноз** | Таблица Prophet на 6 месяцев с доверительными интервалами |
| **График** | Две редактируемые линейные диаграммы Excel (топ-N и все запросы) |
| **Голубые океаны** | 3 блока: низкая конкуренция + рост (momentum / mean / slope) |
| **Растущие рынки** | 3 блока: рейтинг по росту (momentum / mean / slope) |

Имя файла результата: `yandex_analysis_ГГГГММДД_ЧЧММСС.xlsx` (в папке с exe или проектом).

## YandexGPT — опционально

Переключатель **«Использовать LLM»** на главной странице **по умолчанию выключен**.

| Режим | Поведение |
|-------|-----------|
| LLM **выкл.** | Полный отчёт только на данных Wordstat, без блоков YandexGPT |
| LLM **вкл.** + ключи GPT | Расширение запросов, обзор, рекомендации, вердикты нейросети |
| LLM **вкл.** + только Wordstat | Ключ и Folder ID Wordstat подставляются для GPT; нужна роль `ai.languageModels.user` |

Отдельные поля YandexGPT в настройках необязательны — можно оставить пустыми и использовать ключ Wordstat.

Проверка GPT: **Настройки → YandexGPT → «Проверить подключение»**.

### Python

```bash
python run.py
```

Откроется **http://127.0.0.1:8000** (браузер запускается автоматически).

Другой порт:


```bash

python run.py
```

### Готовый exe

После сборки запустите:

```
dist\yandex_wordstat_agent.exe
```

Достаточно **одного файла exe** — шаблон Excel и интерфейс встроены внутрь. Отдельный `yandex_analysis_exemple.xlsx` рядом с exe не нужен.

## Сборка exe

```powershell
powershell -ExecutionPolicy Bypass -File .\build.ps1
```

Скрипт:
1. Останавливает старые процессы `yandex_wordstat_agent.exe`
2. Собирает `dist\yandex_wordstat_agent.exe` через PyInstaller
3. Вкладывает `yandex_analysis_exemple.xlsx` в exe (если файл есть в корне проекта)

Перед повторной сборкой закройте запущенный агент или снова используйте `build.ps1`.

## Использование

1. Откройте веб-интерфейс (`http://127.0.0.1:8000` или после запуска exe).
2. **Настройки** → укажите **API-ключ** и **Folder ID** Yandex Cloud (Wordstat).
3. При необходимости настройте Telegram / VK / Email / прокси / YandexGPT.
4. На главной:
   - введите запросы (через запятую или с новой строки);
   - выберите период;
   - при необходимости включите **«Использовать LLM»**;
   - нажмите **«Запустить анализ»**.
5. Скачайте Excel или отправьте отчёт в выбранный канал.

Расширение запросов от YandexGPT (если включён LLM) записывается в Excel **построчно** — можно скопировать и вставить в поле запросов.

## Как правильно заполнить вкладку Настройки

Все поля сохраняются в зашифрованный `settings.enc`. После изменения паролей и токенов нажмите **«Сохранить»**.

### Yandex Wordstat (обязательно)

| Поле | Что указать |
|------|-------------|
| **API-ключ** | Сервисный ключ Yandex Cloud |
| **Folder ID** | ID каталога, в котором включён Wordstat |

Без этих двух полей анализ не запустится. Ключ и каталог создаются в [console.cloud.yandex.ru](https://console.cloud.yandex.ru); сервисному аккаунту нужна роль `search-api.executor`.

### YandexGPT (необязательно)

| Поле | Что указать |
|------|-------------|
| **API-ключ YandexGPT** | Можно оставить пустым — подставится ключ Wordstat |
| **Folder ID для GPT** | Можно оставить пустым — подставится Folder ID Wordstat |
| **Модель** | `yandexgpt-lite`, `yandexgpt` или `yandexgpt-pro` |

В каталоге нужна роль `ai.languageModels.user`. Проверка: кнопка **«Проверить подключение»** в этом разделе. На главной странице переключатель **«Использовать LLM»** по умолчанию **выключен** — включайте только если нужны блоки нейросети в отчёте.

### Telegram

В Telegram **два поля** — это нормально. Отдельных «от кого / кому», как в email, нет: отправитель задаётся токеном бота, получатель — Chat ID.

| Поле | Что указать |
|------|-------------|
| **Bot Token** | Токен от [@BotFather](https://t.me/BotFather), формат `1234567890:AAH...` |
| **Chat ID** | **Ваш личный числовой ID** — кому придёт файл (аналог «Кому» в email) |

**Важно:**

- В Chat ID указывается **ваш user_id**, а **не** ID бота и **не** `@username`.
- Перед отправкой **напишите боту `/start`** в Telegram.
- Свой ID можно узнать через [@userinfobot](https://t.me/userinfobot) или [@getmyid_bot](https://t.me/getmyid_bot).
- В России часто нужен **VPN** или **прокси** (раздел «Прокси»): ошибка `Не удалось подключиться к api.telegram.org` означает блокировку сети, а не неверный Chat ID.

**Пример рабочей схемы:** токен бота + Chat ID `77941560` (ваш личный ID).

### VK

Поддерживаются **два варианта** — токен пользователя или ключ сообщества.

#### Вариант А: ключ сообщества (проверено, рабочий)

| Поле | Что указать | Пример |
|------|-------------|--------|
| **Access Token** | Ключ доступа сообщества (`vk1.a....`) | из Управление → Работа с API |
| **ID сообщества** | Положительное число из ссылки `vk.com/clubXXXXX` | `236999457` для [club236999457](https://vk.com/club236999457) |
| **Peer ID** | Числовой **user_id получателя** (кому в личку) | `77941560` — **не** ID сообщества |

**Права ключа:** «Сообщения сообщества» и «Документы».

**В группе:** Управление → Сообщения — сообщения включены. Желательно один раз написать сообществу в личку (иначе VK может вернуть ошибку 901).

**Частая ошибка:** подставить ID сообщества в поле Peer ID — так делать нельзя. Peer ID — это ID **человека**, которому уходит отчёт.

#### Вариант Б: OAuth-токен пользователя

| Поле | Что указать |
|------|-------------|
| **Access Token** | Токен пользователя, scope `docs,messages,offline` |
| **ID сообщества** | Оставить пустым |
| **Peer ID** | Ваш user_id или ID диалога |

### Email (SMTP)

| Поле | Что указать |
|------|-------------|
| **SMTP хост** | Адрес сервера |
| **Порт** | Обычно `587` (STARTTLS) или `465` (SSL) |
| **Логин** | Полный email |
| **Пароль** | Пароль от почтового ящика |
| **От кого (From)** | Тот же полный email, что и логин |
| **Кому (To)** | Адрес получателя |
| **Использовать TLS** | Включено для порта 587 |

#### Проверенные настройки Disroot

| Поле | Значение |
|------|----------|
| SMTP хост | `disroot.org` |
| Порт | `587` + TLS **включён** (или `465`) |
| Логин | `you@disroot.org` |
| От кого | `you@disroot.org` (совпадает с логином) |
| Пароль | Тот же, что на [mail.disroot.org](https://mail.disroot.org) |

Проверка без отправки файла: кнопка **«Проверить SMTP»** (сначала сохраняет настройки, затем проверяет вход).

Ошибка **535** — сервер отклонил логин или пароль (TLS при этом обычно уже работает). Перепроверьте пароль на веб-почте, заново введите его в настройках и нажмите **«Сохранить»**.

### Прокси

| Поле | Когда нужно |
|------|-------------|
| **Тип / хост / порт** | Если Telegram не подключается к `api.telegram.org` без VPN |

Типы: HTTP или SOCKS5. Для VK и Email прокси обычно не требуется.

### Общие настройки

| Поле | Назначение |
|------|------------|
| **TOP-N** | Сколько запросов в топ-графиках Excel и на главной |
| **Задержка между запросами** | Пауза между обращениями к Wordstat API (по умолчанию 0,4 с) |

### Краткая шпаргалка по проверенным каналам

| Канал | Статус | Что нужно |
|-------|--------|-----------|
| **Email (Disroot)** | ✅ Работает | `disroot.org:587`, TLS, логин = From = `you@disroot.org` |
| **VK (сообщество)** | ✅ Работает | Токен группы + ID `236999457` + peer_id `77941560` |
| **Telegram** | ⚠️ Нужен VPN/прокси в РФ | Токен бота + **ваш** Chat ID (не ID бота), `/start` боту |

| Сервис | Где получить | Роль / примечание |
|--------|--------------|-------------------|

## Ограничения

- **Wordstat API** — лимиты тарифа Yandex Cloud (на бесплатном уровне — порядка 100 запросов/сутки).
- **Период** — до 3 лет (ограничение API).
- **Prophet** — нужно ≥12 месяцев истории по запросу.
- **Запросы без данных** — попадают в сводку и таблицы ниш с нулевыми метриками.
- **Прокси** — настраивается в разделе настроек при необходимости.

## Устранение неполадок

| Проблема | Решение |
|----------|---------|
| Exe не пересобирается | Закройте агент; запустите `build.ps1` или `scripts\stop_agent.ps1` |
| Exe не стартует на другом ПК | Установите [VC++ Redistributable x64](https://aka.ms/vs/17/release/vc_redist.x64.exe) |
| «Настройки API не заполнены» | Укажите ключ и Folder ID Wordstat в настройках |
| Нет прогноза | Мало истории (<12 мес.) или Prophet недоступен |
| YandexGPT не отвечает | Проверьте роль `ai.languageModels.user`; кнопка «Проверить» в настройках |
| Порт занят | Закройте старый экземпляр; при запуске exe предыдущий процесс на порту завершается автоматически |
